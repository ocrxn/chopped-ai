How it Works



Step 1 Media Upload

Users upload media in the form of a video file (separate audio file acceptable) which must include the spoken keywords for each play.

Step 2 Media to Backend

Uploaded files are moved to the backend folder where they are stored and prepared for processing.

Step 3 Extracting Key Events

Detect audio corresponding to keywords and pair them with the timestamp where they occurred.

Step 4 Match Files

Event data is matched with the corresponding video file to ensure that detected key moments are applied to the correct game footage.

Step 5 Cut Clips

Segment video into clips based on detected key event timestamps.

Step 6 Return Clips to User

Generated clips are packaged and returned to the user through the web interface.
